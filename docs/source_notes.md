# Source And Method Notes

## Audience Contract

Report shape follows the executive-report structure:

1. Title
2. Executive summary
3. Key findings with visual evidence
4. Recommended next steps
5. Further questions
6. Caveats and assumptions

The visible report merges findings and implications section-by-section so a reader can audit the conclusion without reading code.

## Data Sources

- Fjelstul World Cup Database CSVs:
  - `tournaments.csv`
  - `qualified_teams.csv`
  - `matches.csv`
  - `tournament_standings.csv`
- FIFA API endpoints:
  - `https://api.fifa.com/api/v3/rankingschedules/all?type=0&gender=1&language=en`
  - `https://api.fifa.com/api/v3/rankingsbyschedule?rankingScheduleId={id}&language=en`

## Derived Artifacts

- `data/derived/ccs_team_year.csv`: team-year CCS flags for World Cups through 2022.
- `data/derived/fifa_rankings_pre_wc.csv`: nearest pre-tournament ranking snapshot for 1998-2026.
- `data/derived/favorite_traps.csv`: Top-12 FIFA-ranked teams that were not CCS candidates at tournament start; retained as the full audit table.
- `data/derived/favorite_trap_headliners.csv`: each tournament's single highest-ranked non-CCS team; used for the main favorite-downgrade exhibit.
- `data/derived/random_benchmark.csv`: random same-size candidate-pool benchmark.
- `data/derived/ccs_2026_watchlist.csv`: 2026 ranked teams with CCS status.

## Visualization QA

Charts are static PNGs generated by `scripts/build_report.py` with Matplotlib. Main charts are generated separately for English and Chinese so labels fit their report language. Each chart has an adjacent interpretation paragraph in the HTML report. The report was rendered to PDF with Chrome headless and inspected through PDF metadata/text extraction.
